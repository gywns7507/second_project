# 💳 Lending Club 부도 예측 모델

**국비교육 심화 프로젝트** | 2026.02.27 | 2026.03.11

---

## 📌 프로젝트 개요

> 미국 최대 P2P 대출 플랫폼 Lending Club의 실제 대출 데이터를 기반으로, 머신러닝 부도 예측 모델을 구축하고 비즈니스 관점의 비용-편익 분석까지 연결한 프로젝트입니다.
> 

**문제 상황**: Lending Club의 누적 부도율 **20%** — 미국 상업은행 신용카드 부도율(약 2%)의 약 10배 수준으로, 불량 차용자로 인한 지속적인 손실 발생

**핵심 목표**: 이자 수익 극대화가 아닌 **원금 손실 최소화** — 부도자를 사전에 식별하는 Recall 중심 모델 설계

**접근 방식**: 대출 신청 시점 정보만 활용 → 데이터 누수 방지 → 머신러닝 분류 모델 → 비용-편익 분석 → 리스크 등급별 정책 제안

- **팀 관리**: 피그마 디자인 관리, 노션 프로젝트 관리, PPT 제작 및 발표

## 🔍 분석 과정

### 1️⃣ 전처리 (Preprocessing)

- **데이터 누수 방지**: 대출 신청~실행 사이 정보만 사용, 실행 이후 발생 변수 전면 제거 (total_pymnt, recoveries 등)
- **변수 선택 4단계 기준**: 누수 위험 변수 → 결측치 80% 이상 → 특정 기간 이후 생성 변수 → 1차 사전 분석(피처 중요도 + 통계 + 비즈니스) 순으로 제거
- **결측치 처리**: 수치형 → 중앙값 대체 / 범주형 → CatBoost 네이티브 처리 + Unknown 범주 지정 / Train-Test 분리 후 Train 기준으로만 처리
- **이상치 보존 근거 3가지**: 금융 극단값은 실제 유효한 정보 / 트리 모델은 극단값에 강건 / 극단값 자체가 부도 예측 신호
- **파생변수 생성**: 결측 FLAG 변수 / 신용점수 평균(mid_fico) / 개인 재무 건전성 지표 3종 (Loan_to_income, Installment_to_income, Revol_bal_to_income)

### 2️⃣ 모델링 (Modeling)

3가지 파이프라인 비교 실험:

| 조합 | PR-AUC | F1-Score | Recall |
| --- | --- | --- | --- |
| SMOTE + XGBoost + GridSearchCV | 0.41 | 0.45 | 0.65 |
| **Class_weight + CatBoost + Optuna** ✅ | **0.42** | **0.45** | **0.64** |
| SMOTE + CatBoost + Analytics Vidhya | 0.41 | 0.45 | 0.71 |

→ 성능 유사, 속도 빠르고 간편한 **Class_weight + CatBoost + Optuna** 최종 선택

**최종 모델 성능** (임계값 0.53 기준)

| 지표 | 값 |
| --- | --- |
| Accuracy | 0.69 |
| Recall (부도) | 0.64 |
| F1-Score (부도) | 0.45 |
| PR-AUC | 0.42 |
| 최적 임계값 (비용-편익 기준) | 0.55 |

### 3️⃣ 통계 분석

- **연속형**: Shapiro-Wilk + QQ-Plot → 비정규성 확인 → Mann-Whitney U Test
    - 이자율(int_rate): 효과 크기 **0.37 (중간 효과)** — 모델이 리스크를 판단하는 가장 강력한 수치 근거
    - 부채비율(DTI): 효과 크기 **0.25 (작은 효과)** — 부채 비율 높을수록 가처분 소득 감소 → 연체 리스크 증가
- **범주형**: Chi-Squared + Cramér's V + Adjusted Residuals
    - 60개월 대출: 기대 대비 부도 빈도 압도적으로 높음 (잔차 +68.4)
    - 2016~2017년 발행 대출: 부도 빈도 유의미하게 높음

### 4️⃣ SHAP 분석 (내 담당)

- **가장 위험한 대출 프로필**: 고이자율 + 60개월 만기 + 높은 DTI 조합
- 대출 발행 연도(issue_year)의 높은 영향력 → 실업률, 금리 등 거시 경제 변수 누락의 구조적 한계 반영
- 근속연수, 주거 형태는 영향력 낮음 → 대출 심사 시 과대평가 지양 필요

## 💬 튜터 피드백 & 개선사항

### ✅ 잘한 점

- 부도율 20%를 수치로 제시하고 **'원금 손실 최소화'라는 명확한 비즈니스 목표** 설정
- 151개 변수를 6개 카테고리로 체계 분류 및 **타임라인 시각화로 데이터 누수 기준 명확히 제시**
- 이상치 미제거 이유를 3가지 논리(유효한 정보 / 트리 강건성 / 예측 신호)로 설득력 있게 설명
- 3가지 파이프라인 비교 실험 설계 체계적, PR-AUC를 불균형 데이터 평가 지표로 적절히 선택
- 혼동 행렬 각 셀에 비즈니스 의미 부여 + 원리금 균등 상환 공식 적용한 **비용-편익 분석이 가장 인상적**
- 임계값 0.53(모델 기준)과 0.55(수익 최적) 거의 일치 → **모델 의사결정의 비즈니스 타당성 검증**
- Current 데이터 활용 → 부도 확률 예측 → 현실 보정 → 리스크 등급 → 대출 정책 **5단계 실무 활용 시나리오** 구성
- ML 프로젝트 전체 사이클(누수 방지 → Feature Engineering → 불균형 처리 → 비용-편익 → SHAP → 통계 → 시나리오)을 높은 수준으로 완성 → **포트폴리오로 충분히 활용 가능**

### 🔧 개선사항

**분석 측면**

- 2012년 이전 데이터 제거 기준에 대한 **구체적 근거 추가** 필요 (단순 컬럼 불일치 외 이유)
- 최종 사용 데이터의 **기간과 건수 명시** 필요 (청중 이해도 향상)
- 교차검증 결과 분산 비교 및 Recall-Precision 트레이드오프 차이 등 **성능 안정성 비교** 추가 시 모델 선택 신뢰성 향상
- Class_weight 선택 근거를 "속도" 외에 **시계열 데이터의 분포 보존 측면**으로 보완 필요
- 이자율(int_rate), 서브그레이드(sub-grade)는 렌딩클럽 심사 파생 변수 → **결과 변수 vs 원본 변수 여부 검토** 필요
- 테스트 셋으로 많은 작업 수행 → **Validation Set 별도 구성** 후 튜닝에 활용, 테스트 셋은 최종 평가에만 사용
- 랜덤 스플릿 대신 **시간 기반 데이터 분할** 적용 필요 (서브프라임 사태 등 경제 변화 반영)

**모델 성능 측면**

- Recall이 생각보다 높지 않음 → **임계값 추가 하향 조정** 또는 cost 기반 최적화 강화 고려
- SMOTE는 금융 데이터에서 극단값 의미를 훼손할 수 있음 → **Class Weight 튜닝 / Focal Loss / Undersampling + Ensemble** 대안 고려
- Feature Engineering 심화 필요: 기본 파생변수 외 **시간 기반 변수(최근 3개월 변화 등) 및 행동 기반 변수** 추가
- SHAP 해석 고도화 필요: '변수 중요도' 수준에서 나아가 **구간별 분석(DTI 40 이상 등) 및 Interaction 분석**으로 "왜 위험한지" 설명 강화
